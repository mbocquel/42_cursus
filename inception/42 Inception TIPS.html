    Quelques mots-clefs Dockerfile:

    FROM

      Permet dindiquer Docker sous quel OS doit tourner votre machine virtuelle.

      Les plus courants sont debian:buster pour Debian ou alpine:x:xx pour Linux.

    RUN

      Permet de lancer une commande sur votre machine virtuelle

      En gnral, les premiers RUN fournit dans le Dockerfile consistent mettre jour les ressources de votre VM, comme apk, ou encore dajouter les utilitaires basiques comme vim, curl ou sudo.

    EXPOSE

      Ici, cest une question de rseau

      L'instruction EXPOSE informe Docker que le conteneur coute sur les ports rseaux spcifis au moment de l'excution. EXPOSE ne rend pas les ports du conteneur accessibles l'hte.

      Attendez ! Quoi ? Le conteneur coute sur le port rseau et n'est pas accessible l'hte ?

      Qu'est-ce que cela signifie ?

      L'instruction EXPOSE expose le port spcifi et le rend disponible uniquement pour la communication entre conteneurs. Comprenons cela l'aide d'un exemple.

      Disons que nous avons deux conteneurs, une application wordpress et un serveur mariadb. Notre application wordpress a besoin de communiquer avec le serveur mariadb pour plusieurs raisons.


      Pour que l'application WordPress puisse parler au serveur MariaDB, le conteneur WordPress doit exposer le port. Jetez un il au Dockerfile de l'image officielle de wordpress et vous verrez une ligne disant EXPOSE3306. C'est ce qui aide les deux conteneurs communiquer l'un avec l'autre.

      Ainsi, lorsque votre conteneur WordPress essaie de se connecter au port 3306 du conteneur MariaDB, c'est l'instruction EXPOSE qui rend cela possible.

      Note : Pour que le serveur WordPress puisse communiquer avec le conteneur MariaDB, il est important que les deux conteneurs soient excuts dans le mme rseau docker.

      Il suffit dindiquer votre commande, argument par argument, dans le format suivant :

    Mais quest ce que TLS ?


    Wikipdia nous indique :

    La Transport Layer Security (TLS) ou  Scurit de la couche de transport  est un protocole de scurisation des changes par rseau informatique, notamment par Internet.


    TLS permet :

    • l'authentification du serveur
    • la confidentialit des donnes changes (ou session chiffre)
    • l'intgrit des donnes changes
    • de manire optionnelle, l'authentification du client (mais dans la ralit celle-ci est souvent assure par la couche applicative)

    Je ne sais pas vous mais TLS me semble ressembler de trs prs a du SSL, ce fameux protocole qui rajoute un cadenas vert quand vous accdez un site scuris, comme la plupart des sites aujourd'hui.


    Ma question est donc rapidement,

    Quelle est la diffrence entre TLS et SSL ?


    SSL et TLS sont deux protocoles qui permettent lauthentification, et le chiffrement des donnes qui transitent entre des serveurs.


    En fait le SSL est le prdcesseur du TLS. Au fil du temps, de nouvelles versions de ces protocoles ont vu le jour pour faire face aux vulnrabilits et prendre en charge des suites et des algorithmes de chiffrement toujours plus forts, toujours plus scuriss


    Lhistoire du SSL/TSL :


    Initialement dvelopp par Netscape, le SSL sort en 1995 dans sa version SSL 2.0 (le SSL 1.0 ntant jamais sorti).

    Mais aprs la dcouverte de plusieurs vulnrabilits  en 1996, la version 2.0 est vite remplace par le SSL 3.0. 



    Bas sur le SSL 3.0, le TLS est introduit en 1999 comme la nouvelle version du SSL.

    Le protocole TSL est donc simplement le remplaant du SSL. Il corrige certaines vulnrabilits de scurit dans les anciens protocoles SSL.


    Nous devrons donc rendre notre systme scuris, par un protocole semblable au SSL, le TSL.
